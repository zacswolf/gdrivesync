OAuth, Picker, and privacy site

Google files to Markdown, Marp, or CSV, with a real sync button.

GDriveSync links Google Docs and DOCX files to normal Markdown files, Google Slides and PowerPoint files to Marp-flavored Markdown, and Google Sheets and XLSX files to local CSV output in VS Code and through an agent-friendly CLI. Sync is one-way by design: Google Drive stays the source of truth. Multiple connected Google accounts are supported for selecting, linking, and recovering access to files when needed. Markdown and Marp outputs can also opt into local OCR or direct provider-based image enrichment without sending image data to a hosted GDriveSync service.

Why this site exists

GDriveSync uses this domain for its public homepage, privacy policy, and hosted Google Picker flow for supported Docs, Slides, Sheets, DOCX, PPTX, and XLSX sources.

What GDriveSync stores

OAuth tokens for connected Google accounts stay in local secret or auth storage. File mappings, bound-account metadata, and sync state live in a workspace-side .gdrivesync.json manifest, while generated Markdown assets, image-enrichment cache files, and spreadsheet CSV folders stay on your machine.

Privacy

The extension only uses Google Drive access to read supported Google files that you explicitly link. Read the full policy on the privacy page.
